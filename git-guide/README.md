# Comprehensive Git Guide

A practical guide to Git for teams new to version control, from installation through advanced workflows.

## For New Users

If you've never used Git before, start with **[Concepts](01-concepts.md)** to understand what Git is and why it matters. Then follow the path below in order.

## Learning Path

1. **[Concepts](01-concepts.md)** — Understand version control, repositories, commits, and the Git model
2. **[Installation](02-installation.md)** — Get Git running on macOS or Linux
3. **[Basic Workflow](03-basics.md)** — Init a repo, commit changes, push to a remote, pull updates
4. **[Branching & Merging](04-branching.md)** — Create branches, switch contexts, merge code, resolve conflicts
5. **[Advanced Features](05-advanced.md)** — Rebase, cherry-pick, stash, squash commits, amend history
6. **[Collaboration Patterns](06-collaboration.md)** — Pull request workflows, code review, team practices
7. **[Fixing Mistakes](07-troubleshooting.md)** — Undo commits, recover lost work, reset history safely
8. **[Command Reference](08-reference.md)** — Quick lookup for common commands and flags

## How to Use This Guide

- **Learning it the first time?** Read the sections in order. Each builds on the previous one.
- **Visual learner?** Jump to [Visual Guide](09-visual-guide.md) to see ASCII diagrams of how the commit graph evolves.
- **Looking for a specific task?** Jump to the relevant section and use the **Quick Start** at the top.
- **Need a command?** Check **[Command Reference](08-reference.md)**.

## Key Principles

This guide emphasizes:
- **Understanding over memorization** — Why Git works a certain way matters more than remembering syntax
- **Safe operations** — We show how to undo mistakes before teaching destructive commands
- **Team workflows** — Real-world practices that scale from solo projects to teams
- **Practical examples** — Every concept includes a concrete example you can try

## Getting Help

- Within a Git command, add `--help`: `git commit --help`
- For interactive learning: `git` (no arguments) shows basic commands
- If something goes wrong, jump to [Fixing Mistakes](07-troubleshooting.md)
