3. **[Basic Workflow](https://roymatsunaga.github.io/git-guide/03-basics/)** — Init a repo, commit changes, push to a remote, pull updates
4. **[Branching & Merging](https://roymatsunaga.github.io/git-guide/04-branching/)** — Create branches, switch contexts, merge code, resolve conflicts
5. **[Advanced Features](https://roymatsunaga.github.io/git-guide/05-advanced/)** — Rebase, cherry-pick, stash, squash commits, amend history
6. **[Collaboration Patterns](https://roymatsunaga.github.io/git-guide/06-collaboration/)** — Pull request workflows, code review, team practices
7. **[Fixing Mistakes](https://roymatsunaga.github.io/git-guide/07-troubleshooting/)** — Undo commits, recover lost work, reset history safely
8. **[Command Reference](https://roymatsunaga.github.io/git-guide/08-reference/)** — Quick lookup for common commands and flags

## Key Principles

This guide emphasizes:
- **Understanding over memorization** — Why Git works a certain way matters more than remembering syntax
- **Safe operations** — We show how to undo mistakes before teaching destructive commands
- **Team workflows** — Real-world practices that scale from solo projects to teams
- **Practical examples** — Every concept includes a concrete example you can try

## Getting Help

- Within a Git command, add `--help`: `git commit --help`
- For interactive learning: `git` (no arguments) shows basic commands
- If something goes wrong, jump to [Fixing Mistakes](07-troubleshooting)
