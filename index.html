    Comprehensive Git Guide¶

    A practical guide to Git for teams new to version control, from installation through advanced workflows.

    Get Started¶

    This guide is organized into progressive sections. Choose your path:

    New to Git?¶

    Start with Concepts to understand what Git is and how it works, then follow through Installation and Basic Workflow.

    Visual Learner?¶

    Jump to the Visual Guide to see ASCII diagrams of how the commit graph evolves through various Git operations.

    Looking for a Specific Topic?¶

    • Branching & Merging — Work with branches, merge code, resolve conflicts
    • Advanced Features — Rebase, cherry-pick, stash, squash commits
    • Collaboration — Pull request workflows, code review, team practices
    • Troubleshooting — Fix mistakes, recover lost work, undo commits
    • Command Reference — Quick lookup for commands and flags

    Full Learning Path¶

    1. Concepts — Understand version control, repositories, commits, and the Git model
    2. Installation — Get Git running on macOS or Linux
    3. Basic Workflow — Init a repo, commit changes, push to a remote, pull updates
    4. Branching & Merging — Create branches, switch contexts, merge code, resolve conflicts
    5. Advanced Features — Rebase, cherry-pick, stash, squash commits, amend history
    6. Collaboration Patterns — Pull request workflows, code review, team practices
    7. Fixing Mistakes — Undo commits, recover lost work, reset history safely
    8. Command Reference — Quick lookup for common commands and flags
    9. Visual Guide — See how the commit graph evolves through Git operations

    Key Principles¶

    This guide emphasizes: - Understanding over memorization — Why Git works a certain way matters more than remembering syntax - Safe operations — We show how to undo mistakes before teaching destructive commands - Team workflows — Real-world practices that scale from solo projects to teams - Practical examples — Every concept includes a concrete example you can try

    Getting Help¶

    • Within a Git command, add --help: git commit --help
    • For interactive learning: git (no arguments) shows basic commands
    • If something goes wrong, jump to Fixing Mistakes
